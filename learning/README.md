# learning/

Journal d'apprentissage sur l'écosystème IA, Claude Code, les workflows
multi-agent et tout ce qui gravite autour. Un chapitre par thème ou
par session d'apprentissage significative.

Ce dossier est :
- **Relisible** à froid (chaque chapitre se tient seul, pas de
  dépendance cachée au contexte d'une session).
- **Évolutif** : on ajoute des chapitres, on corrige les anciens quand
  quelque chose change.
- **Exportable en PDF** plus tard (structure Markdown propre, titres
  hiérarchisés, pas de dépendance à un rendu exotique).

## Convention d'écriture

- Un chapitre = une session dense d'apprentissage, pas un résumé fourre-tout.
- Titre numéroté pour garder l'ordre chronologique.
- Dans chaque chapitre : une **table des matières** en tête, des
  sections claires, des encadrés "**À retenir**" ou "**Anti-pattern**"
  quand c'est structurant.
- Pas d'abréviation non définie. Si c'est du jargon IA/dev, on le
  définit à la première occurrence.
- Exemples concrets > explications abstraites.

## Export PDF (plus tard)

Quand il y aura assez de matière, on concaténera les chapitres et on
produira un PDF via pandoc ou équivalent. La structure Markdown
actuelle le permet sans retravail.
